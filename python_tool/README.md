# BLE 配网测试上位机 (Python)

这是一个基于 Python 和 `bleak` 库开发的简易 BLE 上位机工具，用于测试 ESP32-S3 BLE 配网组件的功能。它可以模拟手机 APP 与 ESP32 设备进行交互，发送控制指令并接收状态反馈。

## 功能特性

*   自动扫描并连接指定名称的 BLE 设备（默认名称：`ESP32-S3-TEXT`）。
*   支持发送 JSON 格式的控制指令。
*   实时接收并解析设备返回的 Notify 通知。
*   提供交互式菜单，支持以下功能：
    *   获取设备系统信息
    *   WiFi 扫描
    *   WiFi 连接（支持输入 SSID 和密码）
    *   查询已保存的 WiFi
    *   删除已保存的 WiFi
    *   远程重启设备

## 环境要求

*   Python 3.7 或更高版本
*   电脑具备蓝牙适配器（并已开启蓝牙功能）
*   操作系统：Windows 10/11, macOS, Linux

## 安装依赖

在使用本工具前，请确保已安装必要的 Python 依赖库。

1.  打开终端或命令行工具。
2.  进入本目录：
    ```bash
    cd components/ble/python_tool
    ```
3.  安装依赖：
    ```bash
    pip install -r requirements.txt
    ```
    或者手动安装：
    ```bash
    pip install bleak
    ```

## 使用方法

1.  **启动脚本**：
    在终端中运行以下命令：
    ```bash
    python ble_test.py
    ```

2.  **自动连接**：
    脚本启动后会自动扫描名为 `ESP32-S3-TEXT` 的设备。请确保您的 ESP32 开发板已上电并处于广播模式。
    *   *提示：如果您的设备名称不是 `ESP32-S3-TEXT`，请修改 `ble_test.py` 中的 `DEVICE_NAME` 变量。*

3.  **交互菜单**：
    连接成功后，终端将显示以下操作菜单：
    ```text
    --- 菜单 ---
    1. 连接 (获取信息)  - 发送连接握手指令，获取设备版本和状态信息
    2. WiFi 扫描       - 请求设备扫描周围 WiFi，结果将通过 Notify 异步返回
    3. WiFi 连接       - 输入 SSID 和密码，让设备连接 WiFi
    4. WiFi 查询       - 查询设备当前已保存的 WiFi 凭据
    5. WiFi 删除       - 删除设备中保存的指定 WiFi 凭据
    6. 重启            - 发送远程重启指令（设备通常会延时 3 秒重启）
    q. 退出            - 断开蓝牙连接并退出程序
    ```

4.  **查看日志**：
    *   **[发送]**: 显示上位机发送给设备的 JSON 指令。
    *   **[通知]**: 显示设备通过 Notify 特征值返回的原始数据。
    *   **[JSON]**: 显示解析后的设备响应数据（格式化后的 JSON）。

## 配置说明

如果您的固件修改了 UUID 或设备名称，请同步修改 `ble_test.py` 顶部的配置项：

```python
# 服务 UUID
SERVICE_UUID = "00000001-0000-1000-8000-00805F9B34FB"
# 命令特征值 UUID (Write)
CHAR_CMD_UUID = "00000002-0000-1000-8000-00805F9B34FB"
# 状态特征值 UUID (Notify)
CHAR_STATUS_UUID = "00000003-0000-1000-8000-00805F9B34FB"

# 目标设备名称
DEVICE_NAME = "ESP32-S3-TEXT"
```

## 常见问题

**Q1: 扫描不到设备？**
*   检查 ESP32 是否已上电且未被其他手机/电脑连接（BLE 通常只允许一个连接）。
*   检查电脑蓝牙是否已开启。
*   确认固件中的广播名称是否为 `ESP32-S3-TEXT`。

**Q2: 报错 `BleakCharacteristicNotFoundError`？**
*   这通常是因为固件中的 GATT 服务表与脚本中定义的 UUID 不匹配，或者某些特征值未正确添加。
*   请检查 ESP32 固件代码 (`ble_gatts_module.c`) 中的 UUID 定义和 `GATTS_NUM_HANDLE_TEST_A`（句柄数量）设置是否足够（建议设置为 8 或更多）。
*   尝试在手机上使用 "nRF Connect" APP 连接设备，查看实际的服务和特征值 UUID。

**Q3: 发送指令无响应？**
*   检查 ESP32 日志（通过 USB 串口查看），确认是否收到了 Write 请求。
*   某些指令（如 WiFi 扫描）是耗时操作，设备会先返回状态，稍后通过 Notify 推送结果，请耐心等待。
